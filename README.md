# Shopee Indonesia Product Scraper
 
A Python scraper targeting Shopee Indonesia (`shopee.co.id`), built as Section 3 of the DE Analytics Portfolio (Kredivo Technical Test).
 
**Status: Partial success.** The scraper is fully functional and successfully collected real product data. Collection was limited by Shopee's multi-layered anti-bot system, documented in full below. `result.csv` and `result.json` contain a combination of real scraped rows and representative sample data demonstrating the full schema.
 
---
 
## How It Works
 
Shopee is a Single Page Application. Its frontend fetches product data from an internal JSON search API (`/api/v4/search/search_items/`). We call that same endpoint directly — no HTML parsing needed.
 
```
scraper.py
  └── build_session()       # curl_cffi session, Chrome 124 TLS impersonation
  └── warm_up_session()     # homepage visit to acquire initial cookies
  └── scrape_all(keywords)
       └── scrape_keyword() # paginates 3 pages × 60 items per keyword
            └── build_url() # constructs API URL with offset params
            └── parse_item()# extracts all fields from JSON response
  └── save_csv() + save_json()
```

## The Anti-Bot Problem — Full Documentation
 
This section documents every layer of Shopee's defenses encountered, the approaches tried, and why each succeeded or failed. This is included because understanding *why* something is hard is as important as solving it.
 
### Layer 1: TLS Fingerprinting
 
**Problem:** Standard Python `requests` is identified at the SSL/TLS handshake level before any headers are read. Shopee reads the cipher suites and extensions of the connection and immediately knows it's not a real browser. Returns 403.
 
**Solution:** Switched to `curl_cffi`, which wraps `libcurl` and copies Chrome 124's exact TLS fingerprint. This makes the connection indistinguishable from a real browser at the network level.
 
**Result:** Partially successful. `curl_cffi` broke through on first contact and collected 37 real product rows from the "sepatu" keyword.
 
### Layer 2: Per-session Rate Limiting
 
**Problem:** After the first successful session, subsequent keyword searches returned 403. Shopee tracks request volume per session and throttles aggressively.
 
**Attempted fix:** Fresh `curl_cffi` session per keyword (new TLS handshake + new cookies from homepage warm-up) + 8–15 second pauses between keywords.
 
**Result:** Inconsistent. Sometimes worked, sometimes 403 immediately. Shopee's rate limiting appears IP-based as well as session-based.
 
### Layer 3: JS-Generated Security Tokens
 
**Problem:** Inspecting the actual browser request headers revealed additional tokens generated by JavaScript on every request:
- `af-ac-enc-dat` — anti-fraud token
- `af-ac-enc-sz-token` — size/session token  
- `x-sap-sec` — 2KB cryptographic signature
- `x-sap-ri` — request integrity token
- `d-nonptcha-sync` — bot detection token
- `sz-token` — session token
These tokens are time-sensitive, URL-specific, and generated by Shopee's client-side JavaScript. They cannot be replicated without executing that JavaScript.
 
**Attempted fix:** Switched to Playwright (Microsoft's browser automation library) to run real Chromium. The browser executes Shopee's JS naturally, generating all tokens automatically. We intercept the API responses as the browser makes them.
 
**Result:** Shopee detected the automated browser and redirected to `/verify/captcha?anti_bot_tracking_id=...` returning a "Terjadi Kesalahan" error page.
 
### What Would Actually Work
 
The only reliable approach without paying for third-party services would be:
- **Residential proxy rotation** — rotates real home IP addresses to avoid IP-based rate limiting
- **Authenticated browser session with CAPTCHA solving** — tools like Camoufox + manual login + CAPTCHA solving service
- **Apify Actor** (`gio21/shopee-scraper`) — managed cloud scraping service with all of the above built in, ~$1 per 1,000 products

## Fields Collected
 
### Mandatory
 
| Field | Type | Description |
|-------|------|-------------|
| `product_name` | string | Product title as listed on Shopee |
| `category_id` | integer | Shopee's numeric category ID |
| `price_idr` | float | Price in IDR (raw value ÷ 100,000) |
 
> Shopee stores prices as integers scaled by 100,000 internally. `parse_price()` handles this conversion.
 
### Extra fields (with business justification)
 
| Field | Type | Business value |
|-------|------|----------------|
| `rating` | float | Demand signal; input to price elasticity models |
| `review_count` | integer | Weights rating reliability (4.9★ from 3 reviews ≠ 4.9★ from 3,000) |
| `units_sold` | integer | Sales velocity; GMV modelling and inventory forecasting |
| `stock` | integer | Live inventory intelligence; triggers restocking alerts |
| `seller_location` | string | Geographic supply mapping; logistics cost analysis |
| `discount_pct` | integer | Promotional strategy; markdown optimisation input |
| `product_url` | string | Full traceability; enables price monitoring across runs |

## How to Run
 
```bash
pip install -r requirements.txt
python scraper.py
```
 
> **Note on live execution:** Due to Shopee's anti-bot system (documented above), live collection may be blocked depending on your IP and network. The scraper implements TLS impersonation, session rotation, and polite delays — but Shopee's JS-token layer and IP rate limiting remain a blocker without residential proxies or authenticated sessions.
 
### Customise
 
All configuration is at the top of `scraper.py`:
 
```python
KEYWORDS               = ["laptop", "smartphone", ...]
PAGES_PER_KEYWORD      = 3       # × 60 items per page
DELAY_BETWEEN_REQUESTS = (1.5, 3.0)  # seconds between pages
```

## Design Decisions
 
**curl_cffi over requests**: Standard `requests` is fingerprinted at the TLS layer. `curl_cffi` impersonates Chrome 124's exact TLS signature. This was the correct technical solution to the identified problem.
 
**Fresh session per keyword**: Shopee rate-limits at the session level. Creating a new session per keyword (new TLS handshake + new cookies) resets that counter and makes each batch look like a new browser visit.
 
**Single file**: The test spec asks for `scraper.py`. Everything is modular internally (single-responsibility functions, typed `@dataclass` model) without needing multiple modules.
 
**`@dataclass` for Product**: Type-annotated, serialisable via `asdict()`. Both `save_csv()` and `save_json()` use the same `asdict()` call with no duplication.
 
**Discount normalisation**: Shopee returns discount as integer (`5`) or string (`"-5%"`). The parser strips `%` and `-` before converting, ensuring consistent integer output.
